# **多模态金融量化挖掘智能体 - 详细技术文档**

## **1. 项目概述**

### **1.1. 核心愿景**

本项目旨在构建一个融合了**生成式人工智能 (Generative AI)** 的逻辑推理能力与**传统量化模型 (Quantitative Models)** 的计算确定性的高级金融决策支持系统。它解决了两大痛点：

1.  **传统量化工具**："有数据，无观点"。它们能提供海量指标和回测结果，但无法像人类分析师一样，将数据转化为结构化、易于理解的投资论点。
2.  **通用大语言模型**："有观点，无数据"。它们能言善辩，但缺乏精确的、实时的金融数据计算能力，容易产生“幻觉”或给出宽泛的建议。

本系统通过“**双脑协同**”架构，让AI智能体（左脑）能够指挥和调度一系列精密的量化工具（右脑），从而实现像资深投资顾问一样思考、分析并给出结构化建议。

### **1.2. 技术栈**

-   **前端框架**: Streamlit
-   **AI 智能体**: DeepSeek V3 (通过 `openai` SDK 调用)
-   **深度学习**: PyTorch, Transformers
-   **数据处理与量化**: Pandas, NumPy, Scipy, `pandas-ta`
-   **数据获取**: `yfinance`

## **2. 系统架构：双脑协同模型**

系统采用模块化设计，核心是“双脑协同”的思想，将推理与计算分离。

```mermaid

    end

    B -->|提问/指令| C;
    E -->|查询工具定义| F;
    E -->|执行具体任务| G;
    E -->|执行具体任务| H;
    E -->|执行具体任务| I;
    E -->|执行具体任务| J;

    G -- 3. 返回计算结果 --> E;

    E -- 4. 汇总工具结果 --> D;
    D -- 5. 综合分析与推理 --> L[报告生成];
    L -- 6. 返回结构化回答 --> B;

```

**工作流程说明:**

1.  **用户交互**: 用户在 `Streamlit` 界面输入查询或点击快捷指令。
2.  **意图理解**: `FinancialAgent` 接收到请求，结合上下文理解用户意图。
3.  **工具规划**: Agent 的决策逻辑模块根据意图，规划需要调用哪些量化工具来获取数据支撑。
4.  **工具调用**: Agent 通过 `tool_registry.py` 查询工具的接口规范，并调用具体的工具类（如 `HybridPredictor`）执行计算。
5.  **数据计算**: 各个量化工具从 `datas/` 目录加载所需数据，执行复杂的数学或模型推理运算，并返回结构化的 JSON 结果。
6.  **结果综合**: Agent 收集所有工具的返回结果。
7.  **生成报告**: Agent 将这些结构化的数据作为新的上下文，调用 `DeepSeek` LLM 进行最后的综合分析、风险识别和结论提炼，生成一份通俗易懂的投资分析报告。
8.  **前端呈现**: 报告以流式对话的形式呈现在用户面前。

## **3. 数据全流程管道 (Data Pipeline)**

从原始数据到模型训练的输入，数据处理流程共分为6个关键步骤，由一系列独立的 Python 脚本按顺序执行。

### **步骤 1: 数据获取 (`get_data.py`)**

-   **输入**: `datas/stock_statistics.csv` (获取股票列表), 命令行参数 (起止日期)。
-   **处理**:
    1.  从 `stock_statistics.csv` 中读取“Percentage”>=0.1的股票代码列表。
    2.  并行下载这些股票的历史日线数据 (OHLCV, Adj Close)。
    3.  同时下载宏观指数数据 (`^VIX`, `^TNX`, `^GSPC`)。
    4.  使用 `yfinance.Ticker().history()` 替代 `yf.download()` 以提高稳定性。
-   **输出**: `datas/stock_datas.csv`，包含所有股票和宏观指数的原始数据。

### **步骤 2: 新闻数据合并 (`merge_news_into_stock_data.py`)**

-   **输入**: `datas/stock_datas.csv`, `datas/analyst_ratings_processed.csv` (新闻数据)。
-   **处理**:
    1.  读取新闻数据，按 (Ticker, Date) 分组。
    2.  将同一天、同一股票下的多条新闻标题用 ` | ` 连接符拼接成一个字符串。
    3.  将聚合后的新闻数据左连接 (left merge) 到 `stock_datas.csv` 上，生成新的 `NewsTitles` 列。
-   **输出**: 更新后的 `datas/stock_datas.csv`。

### **步骤 3: 特征工程 (`feature_engineering.py`)**

-   **输入**: `datas/stock_datas.csv`。
-   **处理**: 这是最核心的数据扩充步骤。
    1.  **宏观特征**: 计算标普500指数 (`^GSPC`) 的对数收益率 `GSPC_LogReturn`。
    2.  **技术指标**: 按 Ticker 分组，独立计算每个股票的多种技术指标，包括：
        -   `LogReturn`: 对数收益率。
        -   `Volatility_20`: 20日对数收益率标准差。
        -   `RSI_14`, `MACD`, `SMA_5`, `SMA_20`, 布林带 (`BBands`), `ATR_14`, `OBV` 等。
    3.  **因子指标**: 计算60日窗口的 `Beta`, `Alpha`, `Sharpe_60`。
    4.  **情感特征**:
        -   加载本地的 `FinBERT` 模型 (`models/yiyanghkust_finbert-tone`)。
        -   对 `NewsTitles` 列中的新闻文本进行情感打分，计算 `Sentiment_Score` (Positive概率 - Negative概率)。
-   **输出**: 更新后的 `datas/stock_datas.csv`，包含了所有计算出的特征列。

### **步骤 4: 数据清洗 (`clean_stock_data.py`)**

-   **输入**: `datas/stock_datas.csv`。
-   **处理**:
    1.  **缺失值填充**: `Sentiment_Score` 填充为0, `NewsTitles` 填充为空字符串。
    2.  **前导 NaN 剔除**: 由于移动平均等指标在数据初期会产生NaN，此步骤会按 Ticker 分组，剔除每个股票开头连续包含NaN的行。
    3.  **异常值处理 (Winsorizing)**: 对收益率、波动率等比率型特征，按 Ticker 分组，将其值限制在1%和99%的分位数之间，以减少极端值对模型训练的干扰。
-   **输出**: 清洗后的 `datas/stock_datas.csv`。

### **步骤 5: 特征筛选 (`vif_feature_selection.py`)**

-   **输入**: `datas/stock_datas_cleaned.csv` (或上一步的输出)。
-   **处理**:
    1.  **解决多重共线性**: 技术指标间通常存在高度相关性 (如 `Close` 和 `SMA_20`)，这会影响模型稳定性和解释性。
    2.  采用**方差膨胀因子 (VIF)** 进行迭代筛选。
    3.  循环计算所有特征的 VIF 值，移除 VIF 最高的特征，然后重复此过程，直到所有特征的 VIF 值都低于阈值 (默认为 5.0)。
-   **输出**: `selected_features.md`，一份详细的Markdown报告，记录了每轮迭代移除的特征和最终保留的18个低相关性特征。

### **步骤 6: VIF 特征过滤 (`clean_stock_data_vif.py`)**

-   **输入**: `datas/stock_datas.csv`, 以及 `vif_feature_selection.py` 中定义的 `VIF_FEATURES` 列表。
-   **处理**: 从清洗后的数据中，只保留基础信息列 (`Date`, `Ticker`, `NewsTitles`) 和 VIF 筛选出的18个特征。
-   **输出**: `datas/stock_datas_vif.csv`，这是用于模型训练的最终、最精简的数据集。

## **4. 核心模型与算法详解 (右脑)**

### **4.1. T+1股价混合预测模型 (`hybrid_model.py`, `hybrid_model_trainer.py`)**

这是项目的旗舰预测模型，采用多任务学习 (Multi-task Learning) 架构。

#### **4.1.1. 目标与输入**

-   **双重目标**:
    1.  **方向准确率 (Directional Accuracy, DA)**: 预测T+1是上涨还是下跌 (二分类任务)。目标列 `Target_DA`。
    2.  **波动幅度 (Magnitude)**: 预测T+1对数收益率的绝对值大小 (回归任务)。目标列 `Target_Magnitude`，并使用了3期指数移动平均进行平滑，以降低噪声。
-   **双模态输入**:
    1.  **标量时序特征**: VIF筛选后的18个数值特征序列。
    2.  **文本特征**: 使用FinBERT提取的`NewsTitles`的768维CLS向量序列。

#### **4.1.2. 模型架构 (`HybridModel`)**

模型由三个核心组件和一个共享层构成：

1.  **Temporal Tower (LSTM)**:
    -   一个2层的 `LSTM` 网络，负责处理18维的标量时序特征，捕捉价格、成交量、技术指标等数据中的时序依赖关系。

2.  **Text Tower (Transformer)**:
    -   一个4层的 `Transformer Encoder`，负责处理768维的文本嵌入序列，捕捉新闻舆情在时间上的演变和内在语义。

3.  **Fusion Layer (Cross-Attention)**:
    -   融合层是关键。它使用**交叉注意力机制 (Cross-Attention)**，将 `Temporal Tower` 输出的向量作为 `Query`，`Text Tower` 输出的所有时间步向量作为 `Key` 和 `Value`。
    -   这允许模型动态地关注与当前市场状态最相关的新闻信息，实现两种模态的智能融合。
    -   设计了一个门控机制 (`Gating Mechanism`) 来平衡原始时序信息和注意力融合后的信息。

4.  **双输出头 (Dual Heads)**:
    -   融合后的特征向量经过一个共享全连接层 (`shared_fc`)。
    -   随后分叉到两个独立的输出头：
        -   **分类头 (`cls_head`)**: `Linear -> Sigmoid`，输出上涨概率 (0-1)。
        -   **回归头 (`reg_head`)**: `Linear`，输出标准化的波动幅度。

#### **4.1.3. 训练过程 (`hybrid_model_trainer.py`)**

-   **损失函数**:
    -   `Loss = 0.5 * BCELoss(DA) + 0.5 * HuberLoss(Magnitude)`
    -   `BCELoss` 用于分类任务。
    -   `HuberLoss` 用于回归任务，它对异常值不敏感，比MSE更稳健。
-   **数据处理**:
    -   使用 `RobustScaler` 对标量特征、文本嵌入和幅度目标进行标准化，以处理异常值。Scaler信息会与模型一同保存。
-   **评估与早停**:
    -   监控验证集上的一个复合损失 `Val Loss = 0.7 * DA Loss + 0.3 * RMSE`。
    -   当这个复合损失在连续15个周期内不再下降时，触发早停，并保存性能最佳的模型。

**当前提供的检查点 (`models/hybrid_model/checkpoints/hybrid_model.pt`) 在验证集的指标：**

-   `Val Loss = 0.71`
-   `Val RMSE = 0.0068`

### **4.2. 支撑/阻力位识别 (`support_resistance.py`)**

-   **算法**: 核密度估计 (Kernel Density Estimation, KDE)。
-   **原理**: 传统支撑/阻力位依赖主观画线或简单的最高/最低点。KDE通过分析历史价格的概率分布，找到**价格密度最高的区域**（即筹码密集峰或交易最频繁的价位）。这些区域构成了最客观的支撑和阻力。
-   **实现**:
    1.  提取过去`N`天（默认252天）的加权价 `(H+L+C)/3`。
    2.  使用 `scipy.stats.gaussian_kde` 拟合价格分布。
    3.  在价格评估空间上找到密度函数的**局部峰值** (`find_peaks`)。
    4.  低于当前价格的峰值为支撑位，高于当前价格的为阻力位。
    5.  每个峰值的密度值代表了该点位的“强度”。

### **4.3. 历史形态相似度搜索 (`similarity_search.py`)**

-   **算法**: 基于Z-Score标准化的时间序列相似度匹配。
-   **原理**: 为了比较K线形态的“形状”而非绝对价格，算法首先对查询序列（如最近20天）和历史滑动窗口序列进行Z-Score标准化，使其均值为0，标准差为1。
-   **实现**:
    1.  提取最近 `query_window` 天的收盘价序列，并进行Z-Score标准化，得到 `target_seq`。
    2.  在历史数据上进行滑动窗口操作，每个窗口长度也为 `query_window`。
    3.  对每个历史窗口序列进行Z-Score标准化。
    4.  计算 `target_seq` 与每个历史窗口序列的**欧氏距离**或**余弦相似度**。
    5.  返回相似度最高的 `top_k` 个历史片段，并统计它们后续 `N` 天的平均涨跌幅作为参考。

### **4.4. 金融情感分析 (`finbert_analyzer.py`)**

-   **模型**: `yiyanghkust/finbert-tone`，一个在金融领域文本上微调过的BERT模型。
-   **功能**:
    1.  **三分类**: 将文本分为 `Positive`, `Negative`, `Neutral`。
    2.  **情感打分 (`calc_score`)**: 返回一个介于-1到+1之间的分数，通常由 `Positive` 概率减去 `Negative` 概率得到。
    3.  **文本嵌入**: `hybrid_model_trainer.py` 利用其底层BERT模型提取CLS Token的768维隐状态向量作为文本特征。

## **5. AI 智能体工作原理 (左脑)**

### **5.1. 决策核心 (`decision_agent.py`)**

`FinancialAgent` 类是整个系统的“大脑”，负责与用户交互和调度工具。

-   **System Prompt**:
    -   通过一个精心设计的系统提示词，为LLM设定角色、能力和行为准则。例如：“你是一位资深的量化金融分析专家...请使用工具...保持客观...”。
-   **工具调用 (Function Calling)**:
    -   利用 `DeepSeek` 模型强大的工具调用能力。`agent` 本身不执行代码，而是生成一个包含函数名和参数的JSON请求。
    -   例如，当用户问“EBAY的支撑位在哪”，LLM会分析后生成类似 `{ "name": "calculate_levels", "arguments": "{'ticker': 'EBAY'}" }` 的请求。
-   **执行循环**:
    1.  **构建消息**: 将系统提示、聊天历史、用户最新问题和当前市场上下文 (`ticker`, `current_price`) 整合为发送给LLM的 `messages`。
    2.  **LLM 推理**: 发送 `messages` 和可用的工具列表 (`tools`) 给 `DeepSeek` API。
    3.  **解析响应**:
        -   如果响应是**文本内容**，说明分析完成，直接返回给用户。
        -   如果响应是**工具调用请求**，进入下一步。
    4.  **执行工具**: `_execute_tool` 方法解析JSON请求，实例化对应的工具类 (如 `SupportResistanceScanner`)，并调用其方法。
    5.  **返回结果**: 将工具的计算结果（一个JSON字符串）封装成一个新的 `role: tool` 消息，追加到 `messages` 中。
    6.  **再次推理**: 将包含工具结果的 `messages` 再次发送给LLM，LLM会基于这些精确数据进行总结和分析，直到最终生成文本回答。

### **5.2. 工具注册表 (`tool_registry.py`)**

-   **作用**: 这是LLM能“看到”的工具菜单。
-   **`tools` 列表**: 严格按照 OpenAI/DeepSeek 的格式，用JSON Schema定义了每个函数的名称、描述和参数。**描述 (`description`) 至关重要**，LLM完全依赖它来判断何时以及如何使用该工具。
-   **`TOOL_MAPPING` 字典**: 将函数名（如 `"calculate_levels"`）映射到实际的Python类（如 `SupportResistanceScanner`），供 `decision_agent.py` 在执行时查找和实例化。

## **6. 前端应用逻辑 (`app.py`)**

`app.py` 使用 `Streamlit` 构建了一个交互式Web界面。

-   **UI布局**:
    -   **侧边栏 (`st.sidebar`)**: 用于全局控制，包括输入API Key、选择股票、日期范围和模拟分析的日期。
    -   **主面板**: 分为三个区域：
        1.  **概览区**: 显示当前分析的股票名称和核心模型信息。
        2.  **可视化区**:
            -   **多模态特征关联图**: 使用 `Plotly` 绘制双Y轴图，将股价与新闻情感得分、聚合市场情绪指数并列展示。
            -   **技术指标图**: 包含RSI、MACD和移动平均线 (SMA) 的子图。
            -   **结构化情报**: 展示模拟日期前5天的关键新闻及其情感得分。
        3.  **聊天机器人区**:
            -   **快捷指令卡片**: 预设了几个常见的复杂分析任务，方便用户一键触发。
            -   **聊天历史**: 显示用户与AI的对话。
            -   **输入框**: 用户可以自由提问。

-   **状态管理 (`st.session_state`)**:
    -   `st.session_state.run_analysis`: 一个布尔标志，在用户点击“启动”按钮后设为 `True`，触发主面板的计算和渲染。
    -   `st.session_state.messages`: 存储当前对话的聊天历史。
    -   `st.session_state.financial_agent`: 缓存 `FinancialAgent` 的实例，避免重复初始化。

-   **数据与缓存**:
    -   `@st.cache_data`: 用于缓存**数据加载和计算结果**，如 `load_stock_data_cached`。当输入参数不变时，直接返回缓存结果，避免重复IO和计算。
    -   `@st.cache_resource`: 用于缓存**资源密集型对象**，如 `load_hybrid_predictor_cached`。模型和Agent实例在整个会话中只加载一次。

## **7. 部署与运行指南**

### **7.1. 环境设置**

1.  **克隆项目**:
    ```bash
    git clone <your-repo-url>
    cd <your-repo-directory>
    ```
2.  **安装依赖**: (建议使用虚拟环境)
    ```bash
    python -m venv venv
    source venv/bin/activate  # on Windows: venv\Scripts\activate
    pip install -r requirements.txt
    ```

### **7.2. 模型与数据准备**

1.  **下载模型文件**:
    -   **FinBERT**: 手动从Hugging Face Hub (`yiyanghkust/finbert-tone`)下载所有文件，并放置于 `models/yiyanghkust_finbert-tone/` 目录下。
    -   **Hybrid Model**: 将预训练好的 `hybrid_model.pt` 和 `hybrid_scalers.npz` 放置于 `models/hybrid_model/checkpoints/` 目录下。如果需要自行训练，请运行 `hybrid_model_trainer.py`。
2.  **执行数据管道**:
    -   **首次运行或更新数据**: 按顺序执行第3节描述的数据处理脚本。
        ```bash
        python get_data.py
        ```
